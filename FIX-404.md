## üìã Common Issues

### "Model not found" in ollama list
The kimi model isn't installed.

**Fix:**
```bash
ollama pull kimi-k2.5-cloud
```

If that doesn't work, the model might have a different name. Try:
```bash
ollama search kimi
```

### Model name has extra stuff
Sometimes models have tags like `:latest` or `:7b`

**Example:** If `ollama list` shows `kimi-k2.5-cloud:latest`, use that EXACT name in settings.

### Proxy issues
Make sure:
- Proxy is running (`node proxy.js`)
- Settings URL is `http://localhost:8080` (not 11434)
